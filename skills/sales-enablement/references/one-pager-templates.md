# 单页资料模板

不同单页资料用例的模板，包含布局指导和文案提示。

## 产品概览单页资料

默认的单页资料。向对您一无所知的人介绍您的产品。

### 结构

```
[Logo]                                              [标语]

标题：用一句话描述您做什么以及为谁服务。

问题所在
2-3 句话描述买家面临的痛点。

解决方案
2-3 句话描述您的产品如何解决它。

为什么选择 [您的产品]
• 差异化因素 1 — 一句话解释好处
• 差异化因素 2 — 一句话解释好处
• 差异化因素 3 — 一句话解释好处

证明
"带有具体结果的客户引言。" — 姓名、职位、公司
[可选：2-3 个指标亮点："X% 改进"、"节省 Y 小时"]

[CTA 按钮/链接]                    [联系方式：name@company.com]
```

### 文案提示

- 标题："用一句话描述您做什么，让人说'告诉我更多'？"
- 问题："在找到您之前，您的买家正在挣扎什么？"
- 差异化因素："如果您只能告诉他们 3 件事，什么会让他们选择您？"

## 特定用例单页资料

针对特定工作流程、垂直领域或问题定制。比产品概览更有针对性。

### 结构

```
[Logo]                                    [用例：例如，"面向销售团队"]

标题：[您的产品]如何帮助 [角色] [实现结果]。

挑战
当 [角色] 需要 [任务] 时，他们面临 [特定痛点]。
这导致 [后果]：[时间浪费 / 金钱损失 / 风险]。

工作原理
1. [步骤 1] — 发生了什么以及为什么重要
2. [步骤 2] — 发生了什么以及为什么重要
3. [步骤 3] — 发生了什么以及为什么重要

结果
• [指标 1]：之前 → 之后
• [指标 2]：之前 → 之后
• [指标 3]：之前 → 之后

客户聚焦
"关于此特定用例的引言。" — 姓名、职位、公司

[CTA："查看实际操作"或"开始试点"]       [联系信息]
```

### 何时使用

- 不同的买家角色需要不同的单页资料
- 行业特定版本（医疗、金融科技、电子商务）
- 用例版本（报告、入职、安全）

## 会后留存资料

旨在强化已经发生的对话。总结您讨论的内容并提出下一步建议。

### 结构

```
[Logo]                                            [会议日期]

会议回顾：[公司名称]

我们讨论的内容
• [他们提到的痛点 1]
• [他们提到的痛点 2]
• [他们试图实现的目标]

[您的产品]如何提供帮助
• [解决痛点 1] — [特定能力或工作流程]
• [解决痛点 2] — [特定能力或工作流程]
• [您如何帮助他们实现目标]

相关证明
"来自类似客户的引言。" — 姓名、职位、公司
[来自类似客户的 1-2 个指标]

建议的下一步
1. [带日期的下一步]
2. [跟进动作]
3. [决策时间表]

[您的姓名]  |  [您的职位]  |  [邮箱]  |  [电话]
```

### 提示

- 在会议后 24 小时内发送
- 引用他们说的具体内容（表明您在倾听）
- 保持建议的下一步具体且有时间限制
- 这是您的倡导者转发给其老板的资料

## 倡导者赋能单页资料

专为您的内部倡导者与他们的团队和领导层分享而设计。旨在让他们看起来聪明。

### 结构

```
[Logo]

为什么我们要评估 [您的产品]

现状
[2-3 句话关于内部挑战，写得好像倡导者
在向他们的团队解释。使用"我们"和"我们的"语言。]

[您的产品]做什么
[1-2 句话。通俗语言，无行话。]

为什么选择这个解决方案
• [原因 1] — 它如何解决我们的具体问题
• [原因 2] — 与我们今天所做的相比如何
• [原因 3] — 与我们评估的替代方案相比如何

预期影响
• [指标]：当前状态 → 预期状态
• [指标]：当前状态 → 预期状态
• [价值实现时间]：在 [X 周] 内上线

还有谁在使用
[他们行业中的 2-3 个知名公司名称]
"相关客户引言。" — 姓名、职位、公司

下一步
• [我们接下来要做什么]
• [我们需要团队提供什么]
• [决策时间表]

有问题？联系 [倡导者姓名] 或 [您的姓名，邮箱地址]。
```

### 为什么这有效

- 用倡导者的语气书写，而不是您的
- 回答他们老板会问的问题
- 包含来自他们尊重的公司的同行证明
- 清晰的请求和时间表以推动内部势头

## 布局指导

### 视觉层次

1. **标题** — 最大字体，页面顶部，立即传达价值
2. **章节标题** — 粗体、清晰，作为可扫描的锚点
3. **正文** — 短句， bullet points 优于段落
4. **证明元素** — 指标和引言应在视觉上突出（更大字体、颜色或标注框）
5. **CTA** — 显眼放置，页面底部或右下角

### 留白

- 边距：所有侧面至少 0.75 英寸
- 章节之间的空间：足以视觉上分隔（不要挤压）
- 如果感觉拥挤，削减内容。永远不要将字体缩小到 9pt 以下。

### 字体大小

| 元素 | 建议大小 |
|---------|---------------|
| 标题 | 18-24pt |
| 章节标题 | 12-14pt 粗体 |
| 正文 | 10-11pt |
| 小字 / 页脚 | 8-9pt |

### 颜色

- 使用品牌颜色作为标题和强调色
- 保持正文深色（黑色或近黑色）在白色背景上
- 限制强调色为 1-2 种以保持视觉一致性
- 使用颜色吸引注意力到指标和 CTA

### 文件格式

- **PDF** 用于电子邮件附件和留存资料
- **Google Slides / PowerPoint** 用于销售代表可自定义的可编辑版本
- 始终包括两者——销售代表会自定义，潜在客户想要干净的 PDF
