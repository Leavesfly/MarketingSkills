# 线索路由规则

决策树、平台特定配置、区域路由、ABM 路由以及快速响应基准。

## 路由决策树

使用此模板映射你的路由逻辑：

```
新线索到达
│
├─ 这是否是指定/目标账户？
│  ├─ 是 → 路由至指定的账户负责人
│  └─ 否 ↓
│
├─ ACV（年度合同价值）是否可能 > $50K？（基于公司规模 + 行业）
│  ├─ 是 → 路由至企业级 AE 团队
│  └─ 否 ↓
│
├─ 这是否是具有团队使用行为的 PLG（产品驱动增长）注册？
│  ├─ 是 → 路由至 PLG 销售专家
│  └─ 否 ↓
│
├─ 线索是否匹配某个区域？
│  ├─ 是 → 路由至区域负责人
│  └─ 否 ↓
│
└─ 默认：在可用销售代表之间轮询分配
   └─ 如果没有可用代表：分配至团队队列，设置 1 小时 SLA
```

根据你的业务定制此决策树。关键原则：**首先路由到最具体的匹配项，然后回退到通用规则。**

## 轮询配置

### 基本轮询规则

1. 将线索均匀分配给符合条件的销售代表
2. 跳过休假、已达容量或管道已满的代表
3. 按配额完成情况进行加权（未达配额的代表获得轻微优先权）
4. 每周或每月重置分配计数
5. 记录每次分配以供审计

### HubSpot 轮询设置

**使用 HubSpot 的轮换工具：**
- 导航至 Automation → Workflows
- 触发器：联系人属性 "Lifecycle Stage" 等于 "MQL"
- 操作：在选定用户之间轮换联系人所有者
- 选项：均匀分配，跳过不可用的所有者
- 在分配后添加延迟 + 任务创建

**使用工作流进行自定义轮换：**
1. 创建自定义属性 "Rotation Counter"（数字类型）
2. 工作流触发器：创建新的 MQL
3. 按轮换计数器值分支（0, 1, 2... 对应每个代表）
4. 将联系人所有者设置为对应的代表
5. 递增计数器（在最大值时重置）
6. 创建带有 SLA 截止时间的跟进任务

### Salesforce 轮询设置

**使用线索分配规则：**
1. Setup → Feature Settings → Marketing → Lead Assignment Rules
2. 按优先级顺序创建规则条目（最具体的优先）
3. 对于轮询：使用分配规则 + 自定义逻辑

**使用 Flow 进行高级路由：**
1. 在线索创建时创建记录触发式 Flow
2. 获取记录：查询自定义的 "Rep Queue" 对象以获取下一个可用代表
3. 决策元素：检查代表的可用性、容量、区域
4. 更新记录：分配线索所有者
5. 创建任务：带有 SLA 的跟进任务
6. 更新 "Rep Queue" 以跟踪上次分配

## 区域路由

### 按地理位置

| 区域 | 地区 | 分配团队 |
|------|------|----------|
| 西部 | CA, WA, OR, NV, AZ, UT, CO, HI | Team West |
| 中部 | TX, IL, MN, MO, OH, MI, WI, IN | Team Central |
| 东部 | NY, MA, PA, NJ, CT, VA, FL, GA | Team East |
| 国际 | 所有非美国地区 | International team |

### 按公司规模

| 细分 | 公司规模 | 团队 |
|------|---------|------|
| SMB | 1-50 名员工 | Inside sales |
| 中型市场 | 51-500 名员工 | Mid-market AEs |
| 企业级 | 501-5000 名员工 | Enterprise AEs |
| 战略级 | 5000+ 名员工 | Strategic account team |

### 按行业

| 垂直领域 | 行业 | 专家 |
|---------|------|------|
| 科技 | SaaS、IT 服务、硬件 | Tech vertical rep |
| 金融 | 银行、保险、金融科技 | Financial vertical rep |
| 医疗 | 医院、制药、健康科技 | Healthcare vertical rep |
| 通用 | 其他所有行业 | General pool (round-robin) |

### 混合区域模型

结合多个维度以实现精确路由：

```
线索到达
├─ 公司规模 > 1000？
│  ├─ 是 → Enterprise team
│  │  └─ 按地理位置子路由
│  └─ 否 ↓
├─ 行业 = Healthcare 或 Financial？
│  ├─ 是 → Vertical specialist
│  └─ 否 ↓
└─ 在通用池中轮询分配
   └─ 按地理偏好加权
```

## 指定账户 / ABM 路由

### 设置

1. **定义目标账户列表**（通常为 50-500 个账户）
2. **在 CRM 中分配账户所有者**（每个账户 1 名代表）
3. **匹配逻辑：** 来自目标账户域名的任何线索都路由至账户所有者
4. **匹配规则：**
   - 电子邮件域名匹配（主要）
   - 公司名称模糊匹配（次要，需要人工审核）
   - IP 到公司的解析（第三级，用于匿名访问者）

### ABM 路由规则

| 层级 | 账户类型 | 路由 | 响应 SLA |
|------|---------|------|----------|
| Tier 1 | 前 20 个战略账户 | 指定所有者，即时警报 | 1 小时 |
| Tier 2 | 前 100 个目标账户 | 指定所有者，标准警报 | 4 小时 |
| Tier 3 | 目标行业/规模匹配 | 区域或轮询 | 同一工作日 |

### 多联系人处理

当同一账户的多个联系人参与时：
- 将所有联系人路由至**同一账户所有者**
- 通知所有者有新联系人进入
- 跟踪账户级别的参与度评分（所有联系人的总和）
- 当来自一个账户的 3+ 联系人参与时，触发 "购买委员会" 警报

## 快速响应数据

### 响应时间对转化的影响

| 响应时间 | 相对资格认定率 | 说明 |
|---------|---------------|------|
| 5 分钟以内 | **21 倍**更可能获得资格认定 | 黄金标准 |
| 5-10 分钟 | 10 倍更可能 | 仍然强劲 |
| 10-30 分钟 | 4 倍更可能 | 对大多数情况可接受 |
| 30 分钟 - 1 小时 | 2 倍更可能 | 低于最佳实践 |
| 1-24 小时 | 基线 | 行业平均水平 |
| 24+ 小时 | 比基线低 60% | 线索实际上已变冷 |

来源：Lead Connect, InsideSales.com

### 实施快速响应

1. **即时通知** — 创建 MQL 时向代表发送推送通知 + 电子邮件
2. **自动任务带计时器** — 创建带有 5 分钟 SLA 倒计时的任务
3. **升级链：**
   - 5 分钟：提醒原始代表
   - 15 分钟：提醒备用代表
   - 30 分钟：提醒经理
   - 1 小时：将线索重新分配给下一个可用代表
4. **衡量和报告** — 每周跟踪实际响应时间；表彰快速响应者

### 快速响应自动化

**触发器：** 创建新的 MQL
**操作：**
1. 通过路由规则分配给代表（即时）
2. 向代表发送推送通知 + 电子邮件
3. 创建任务："联系 [Lead Name] — 5 分钟 SLA"
4. 启动 SLA 计时器
5. 如果 15 分钟内没有记录活动 → 提醒备用代表
6. 如果 30 分钟内没有活动 → 提醒经理
7. 如果 60 分钟内没有活动 → 通过轮询重新分配

### 衡量快速响应

每周跟踪这些指标：
- **首次联系的平均时间**（从 MQL 创建到首次呼叫/电子邮件）
- **首次联系的中位数时间**（受异常值影响较小）
- **在 SLA 内联系的线索百分比**（目标：90%+）
- **按一天中不同时段的联系率**（识别覆盖缺口）
- **按响应时间的转化率**（证明速度的 ROI）
