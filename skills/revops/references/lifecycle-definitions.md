# 生命周期阶段定义

潜在客户生命周期阶段、按业务类型划分的 MQL 标准、SLA 以及拒绝/回收工作流的完整模板。

## 阶段模板

### 订阅者

**进入标准：**
- 选择接收博客、新闻通讯或内容更新
- 不需要公司信息

**退出标准：**
- 通过表单或丰富提供公司信息
- 在一次会话中访问 3+ 页面
- 下载受限内容

**负责人：** 营销（自动）

**进入时的操作：**
- 添加到新闻通讯培育
- 开始跟踪参与分数

---

### 潜在客户

**进入标准：**
- 已识别的联系人，具有姓名 + 电子邮件 + 公司
- 可能来自表单填写、丰富或导入

**退出标准：**
- 达到 MQL 阈值（适配 + 参与）
- 由营销/SDR 手动确认资格

**负责人：** 营销

**进入时的操作：**
- 丰富联系人数据（公司规模、行业、角色）
- 开始评分
- 添加到相关培育序列

---

### MQL（营销合格潜在客户）

**进入标准：**
- 满足适配分数阈值和参与分数阈值
- 或触发高意向行动（演示请求、定价页面 + 表单填写）

**退出标准：**
- 销售接受（成为 SQL）
- 销售拒绝（带有原因代码回收到培育）
- SLA 内无响应（升级到经理）

**负责人：** 营销 → 销售（交接）

**进入时的操作：**
- 向分配的销售代表发送即时警报
- 创建具有 4 小时 SLA 的跟进任务
- 暂停营销培育序列
- 记录所有最近活动以供销售上下文

---

### SQL（销售合格潜在客户）

**进入标准：**
- 销售代表已进行资格对话
- 确认：预算、权限、需求或时间线（至少 4 项中的 2 项）

**退出标准：**
- 创建具有预计价值的机会
- 取消资格（带有原因代码回收）

**负责人：** 销售（SDR 或 AE）

**进入时的操作：**
- 在 CRM 中更新生命周期阶段
- 如果 SDR 确认资格则通知 AE
- 如果尚未对话则开始销售序列

---

### 机会

**进入标准：**
- 在 CRM 中创建正式机会
- 分配交易价值、截止日期和阶段

**退出标准：**
- 成交或流失

**负责人：** 销售（AE）

**进入时的操作：**
- 添加到管道报告
- 创建交易任务（提案、演示等）
- 如果交易可能成交则通知 CS

---

### 客户

**进入标准：**
- 成交交易
- 合同签署并设置付款条款

**退出标准：**
- 流失、扩展或续费

**负责人：** 客户服务 / 账户管理

**进入时的操作：**
- 触发入职序列
- 分配 CS 经理
- 安排启动电话
- 从所有销售序列中移除

---

### 倡导者

**进入标准：**
- NPS 分数 9-10，或积极的推荐行为
- 同意案例研究、推荐语或推荐计划

**退出标准：**
- 持续的项目参与

**负责人：** 客户服务 + 营销

**进入时的操作：**
- 添加到倡导计划
- 请求案例研究或推荐语
- 邀请加入推荐计划
- 在营销活动中展示（经许可）

---

## 按业务类型划分的 MQL 标准模板

### PLG（产品驱动增长）

**适配分数（40% 权重）：**

| 属性 | 分数 |
|-----------|--------|
| 公司规模 10-500 | +15 |
| 公司规模 500-5000 | +20 |
| 目标行业 | +10 |
| 决策者角色 | +15 |
| 使用互补工具 | +10 |

**参与分数（60% 权重）— 高度重视产品使用：**

| 信号 | 分数 |
|--------|--------|
| 创建免费账户 | +15 |
| 完成入职 | +20 |
| 使用核心功能 3+ 次 | +25 |
| 邀请团队成员 | +20 |
| 达到使用限制 | +15 |
| 访问定价页面 | +10 |

**MQL 阈值：** 65 分

---

### 销售驱动（企业级）

**适配分数（60% 权重）— 高度重视适配：**

| 属性 | 分数 |
|-----------|--------|
| 公司规模 500+ | +20 |
| 目标行业 | +15 |
| VP+ 职位 | +20 |
| 确认预算权限 | +15 |
| 使用竞争对手产品 | +10 |

**参与分数（40% 权重）：**

| 信号 | 分数 |
|--------|--------|
| 请求演示 | +25 |
| 参加网络研讨会 | +10 |
| 下载白皮书 | +10 |
| 访问定价页面 2+ 次 | +15 |
| 与销售电子邮件互动 | +10 |

**MQL 阈值：** 70 分

---

**适配分数（50% 权重）：**

| 属性 | 分数 |
|-----------|--------|
| 公司规模 50-1000 | +15 |
| 目标行业 | +10 |
| 经理+ 职位 | +15 |
| 目标地理位置 | +10 |

**参与分数（50% 权重）：**

| 信号 | 分数 |
|--------|--------|

## SLA 模板

### MQL 到 SQL SLA

| 指标 | 目标 | 升级 |
|--------|--------|------------|
| 首次联系尝试 | 4 个工作小时内 | 4 小时时提醒销售经理 |
| 资格决定 | 48 小时内 | 48 小时时自动升级 |
| 安排会议（如果合格） | 5 个工作日内 | 每周管道审查标记 |

### SQL 到机会 SLA

| 指标 | 目标 | 升级 |
|--------|--------|------------|
| 完成发现电话 | SQL 后 3 个工作日内 | 提醒 AE 经理 |
| 创建机会 | SQL 后 5 个工作日内 | 管道审查标记 |

### 机会到成交 SLA

| 指标 | 目标 | 升级 |
|--------|--------|------------|
| 交付提案 | 演示后 5 个工作日内 | AE 经理警报 |
| 交易中停滞在阶段 | 该阶段平均天数的 2 倍 | 管道审查标记 |
| 截止日期推迟 2+ 次 | 立即 | 需要预测审查 |

## 潜在客户拒绝和回收

### 拒绝原因代码

| 代码 | 原因 | 回收操作 |
|------|--------|----------------|
| **FIT-01** | 公司太小 | 培育；如果公司增长则重新评分 |
| **FIT-02** | 行业错误 | 归档；不回收 |
| **FIT-03** | 角色错误/无权限 | 培育；监控组织变化 |
| **ENG-01** | 3 次尝试后无响应 | 90 天后回收到培育 |
| **ENG-02** | 感兴趣但时机不佳 | 回收到培育；60 天后重新参与 |
| **QUAL-01** | 无预算 | 90 天后回收到培育 |
| **QUAL-02** | 使用竞争对手，已锁定 | 回收；在合同续签前触发 |
| **QUAL-03** | 不是真实项目 | 归档；不回收 |

### 回收工作流

1. 销售拒绝 MQL 并提供原因代码
2. CRM 将生命周期阶段更新为"回收"
3. 潜在客户进入回收培育序列（与原始培育不同）
4. 参与分数重置为基线（保留适配分数）
5. 如果潜在客户重新参与并跨越 MQL 阈值，重新路由到销售并标记为"回收 MQL"
6. 单独跟踪回收 MQL 转化率

### 回收培育序列

- **频率：** 双周或每月（低于初始培育的频率）
- **内容：** 行业见解、案例研究、产品更新
- **持续时间：** 6 个月，如果无参与则归档
- **重新 MQL 触发器：** 高意向行动（演示请求、重新访问定价页面）
