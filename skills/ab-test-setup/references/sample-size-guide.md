## 样本量基础

### 所需输入

1. **基线转化率**：你当前的转化率
2. **最小可检测效应（MDE）**：值得检测的最小变化
3. **统计显著性水平**：通常为 95%（α = 0.05）
4. **统计功效**：通常为 80%（β = 0.20）

### 这些参数的含义

**基线转化率**：如果你的页面转化率为 5%，那就是你的基线。

**MDE（最小可检测效应）**：你关心检测的最小改进。根据以下因素设定：
- 业务影响（5% 的提升有意义吗？）
- 实施成本（值得投入精力吗？）
- 现实预期（过去的测试显示了什么？）

**统计显著性（95%）**：意味着观察到的差异由随机因素导致的概率小于 5%。

**统计功效（80%）**：意味着如果存在大小为 MDE 的真实效应，你有 80% 的概率检测到它。

## 持续时间计算器

### 公式

```
持续时间（天）= （每个变体的样本量 × 变体数量）/ （每日流量 × 曝光比例）
```

### 示例

**场景 1：高流量页面**
- 需要：每个变体 10,000（2 个变体 = 总共 20,000）
- 每日流量：5,000 访客
- 100% 曝光于测试
- 持续时间：20,000 / 5,000 = **4 天**

**场景 2：中等流量页面**
- 需要：每个变体 30,000（总共 60,000）
- 每日流量：2,000 访客
- 100% 曝光
- 持续时间：60,000 / 2,000 = **30 天**

**场景 3：低流量且部分曝光**
- 需要：每个变体 15,000（总共 30,000）
- 每日流量：500 访客
- 50% 曝光于测试
- 有效每日流量：250
- 持续时间：30,000 / 250 = **120 天**（太长了！）

### 最短持续时间规则

即使样本量充足，测试也应至少运行：
- **完整 1 周**：捕捉一周内不同天的变化
- **2 个业务周期**：如果是 B2B（工作日与周末模式）
- **跨越发薪日**：如果是电商（月初/月末）

### 最长持续时间指南

避免运行超过 4-8 周的测试：
- 新奇效应消退
- 外部因素介入
- 其他测试的机会成本

## 多变量调整

当有超过 2 个变体（A/B/n 测试）时，你需要更多样本：

| 变体数量 | 乘数 |
|----------|------------|
| 2 (A/B) | 1x |
| 3 (A/B/C) | ~1.5x |
| 4 (A/B/C/D) | ~2x |
| 5+ | 考虑减少变体 |

**为什么？** 更多比较会增加假阳性的机会。你在比较：
- A vs B
- A vs C
- B vs C（有时）

应用 Bonferroni 校正或使用自动处理此问题的工具。

## 常见样本量错误

### 1. 功效不足的测试
**问题**：样本量不足以检测现实效应
**修复**：对 MDE 保持现实态度，获取更多流量，或不进行测试

### 2. 功效过高的测试
**问题**：已经达到显著性却仍在等待样本量
**修复**：这实际上没问题——你已承诺样本量，应遵守它

### 3. 错误的基线率
**问题**：使用错误的转化率进行计算
**修复**：使用特定指标和页面，而非全站平均值

### 4. 忽略细分
**问题**：按全部流量计算，然后分析细分
**修复**：如果计划细分分析，按最小细分计算样本量

### 5. 测试太多内容
**问题**：流量被分割得太细
**修复**：无情地确定优先级，运行更少的并发测试

## 当样本量要求过高时

当无法获得足够流量时的选项：

1. **增加 MDE**：接受只检测更大的效应（20%+ 提升）
2. **降低置信度**：使用 90% 而非 95%（有风险，需记录）
3. **减少变体**：仅测试最有希望的变体
4. **合并流量**：在多个相似页面上测试
5. **测试上游**：在流量更高的漏斗早期测试
6. **不测试**：基于定性数据做出决策
7. **更长测试**：接受更长的持续时间（数周/数月）

## 序贯测试

如果必须在达到样本量之前检查结果：

### 是什么？
一种统计方法，针对多次查看数据进行调整。

### 何时使用
- 高风险变更
- 需要提前停止不良变体
- 时间敏感的决策

### 支持的工具
- Optimizely（Stats Accelerator）
- VWO（SmartStats）
- PostHog（贝叶斯方法）

### 权衡
- 更灵活地提前停止
- 略大的样本量要求
- 更复杂的分析
