# 定位变体库

目录受众对不同的框架反应不同。切勿在所有地方复制粘贴相同的描述 — AI 引擎会惩罚重复内容，且每种目录类型奖励不同的开场白。

使用此库为每个层级生成变体。将 `[product]`、`[category]`、`[competitors]`、`[use-case]` 和 `[audience]` 替换为实际值。

---

## 框架：引导句因层级而异

| 层级 | 引导句模式 | 原因 |
|---|---|---|
| 初创公司 / 发布 | "[Product] 是 [audience] 实现 [outcome] 的最简单方式。" | 创始人扫描结果清晰度。 |
| SaaS 目录 | "[Product] 是 [competitors] 的 [differentiator] 替代品。" | 捕获"[competitor] 替代品"搜索意图。 |
| AI 目录 | "[Product] 使用 [AI capability] 实现 [outcome]。" | TAAFT/Futurepedia 受众明确要求 AI。 |
| Agent / MCP | "[Product] 是 MCP-native / agent-native [category]。" | 小众但高意向。排除竞争对手。 |
| 无代码 | "[Product] 让你无需代码即可构建 [output]。" | 受众重视速度，而非技术深度。 |
| 开发者工具 | "[Product] 是具有 [differentiator] 的 [technical category]。" | 开发者希望 upfront 看到实质内容。 |
| B2B 评论 | "[Product] 帮助 [audience] 实现 [measurable business outcome]。" | 评论者希望看到 ROI 语言。 |

## 模板：初创公司 / 发布目录

**目标：** Product Hunt, BetaList, Fazier, Uneed, DevHunt, Microlaunch, OpenHunts, LaunchVault, Firsto, PitchWall

**标语（少于 10 个词）：**
> 面向 [audience] 实现 [outcome] 的 [differentiator] 方式。

**简短描述（60 字符）：**
> [以结果为导向的一句话，包含产品名称]

**长描述（150 词）：**
> [Product] 是 [audience] 实现 [outcome] 的最简单方式。专为 [pain point] 的团队打造，[product] 通过 [how] 消除 [friction]。
>
> 与 [competitor category] 不同，[product] [key differentiator 1] 和 [key differentiator 2]。你可以在 [timeframe] 内 [action 1]，无需 [limitation] 即可 [action 2]，以及 [action 3]，这通常需要 [cost or Technical skill]。
>
> 我们构建 [product] 是因为 [founder origin story in one sentence]。现在被 [audience examples] 用于 [use case examples]。
>
> 在 [url] 免费试用。无需信用卡，无需设置。

**标签：** [product category], [audience type], [use case 1], [use case 2], [differentiator], [tech]

## Do / Don't 快速参考

**DO：**
- 在不同层级间变化开场句
- 使用真实数字和具体的差异化因素
- 语气匹配受众（开发者用技术性，G2 用商业性，PH 用兴奋感）
- 在初创公司目录中包含创始人/起源角度
- 在 AI 目录中以 AI-first 角度为主导

**DON'T：**
- 在所有地方复制粘贴相同的 150 词描述
- 使用模糊声明（"blazing fast"、"game-changing"）
- 提及每个功能 — 每个层级选择 3–5 个并轮换
- 谎报竞争对手功能（AI 引擎会交叉引用并降权）
- 跳过标签列表 — 这是版主将你路由到正确类别的方式
