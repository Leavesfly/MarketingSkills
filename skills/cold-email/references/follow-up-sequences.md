# 跟进邮件序列

55% 的回复来自跟进邮件，而非初始邮件。然而，48% 的销售人员从未进行过任何跟进。

## 邮件数量：总共 3–5 封

- 单封邮件的最高回复率：**8.4%**（Belkins）。
- 4–7 封邮件的活动可实现 **27% 的回复率**，而 1–3 封邮件仅为 9%（Woodpecker，2000 万封邮件数据）。
- 到第 4 次跟进时，响应率下降 **55%**，垃圾邮件投诉**增加两倍**。
- 解决方案：更长的序列可以覆盖不同的时间窗口。最多进行 4 次跟进（共 5 封邮件）。每封邮件都必须提供真正新的价值。

## 最佳节奏

逐步增加每次联系之间的间隔：

| 联系次数     | 天数  | 说明                                          |
| ------------- | ----- | ---------------------------------------------- |
| 初始邮件 | 0     | 投入最大程度的个性化             |
| 跟进 1   | 3     | 等待 3 天可将响应率提高多达 31% |
| 跟进 2   | 7–8   | 不同角度                                |
| 跟进 3   | 14    | 新的价值内容                                |
| 跟进 4   | 21–28 | 结束邮件                                  |

**最佳日期：** 周二至周四（周四的回复率达到峰值 6.87%）。
**最佳时间：** 潜在客户当地时间的上午 9–11 点或下午 1–3 点。
**避免：** 周一早晨（收件箱过载）、周五下午（心不在焉）。

## 角度轮换

每封跟进邮件必须能够独立存在，同时朝着目标推进。切勿只是"顶一下这封邮件"。

| 邮件       | 角度                                                      | 目的                    |
| ----------- | ---------------------------------------------------------- | -------------------------- |
| 初始     | 个性化钩子 + 核心价值主张 + 温和的 CTA             | 介绍问题/解决方案 |
| 跟进 1 | 不同角度，新的价值内容（统计数据、见解、资源） | 展示额外益处    |
| 跟进 2 | 来自类似公司的社会证明 / 案例研究             | 建立可信度          |
| 跟进 3 | 新见解、行业趋势或相关资源          | 展示专业知识      |
| 跟进 4 | 结束 — 承认沉默，保持开放             | 触发损失厌恶      |

每封邮件只添加**一个新的价值主张**（SalesBread）。这自然会促使采用不同的角度。

## 结束邮件

利用损失厌恶心理 — 通过撤回来减轻压力，同时制造稀缺感。Close.com 报告称，针对冷潜在客户的结束邮件可实现 **10–15% 的响应率**。

**结构：**

1. 承认你已多次联系
2. 理解他们可能缺乏兴趣
3. 声明这是你目前的最后一封邮件
4. 保持大门敞开

**示例：**

> 我没有收到回复，所以假设现在不是合适的时机。在我结束沟通之前：[一句话见解或资源]。如果这改变了你的想法，欢迎回复。否则，没有关系 — 祝你在 [他们的目标] 方面顺利。

**1-2-3 格式**（将摩擦降至接近零）：

> 由于我没有收到回复，我会保持简洁。回复一个数字：
>
> 1 — 感兴趣，我们聊聊
> 2 — 现在不行，3 个月后再联系
> 3 — 不感兴趣，请停止联系

**关键规则：** 如果你发送了结束邮件，请遵守承诺。不要再联系该潜在客户。

## 降低回复率的措辞

- "我从未收到回复" → 会议预订率**下降 12%**（Gong）
- "只是检查一下" → 零价值，表明懒惰
- "把这封邮件顶到你的收件箱顶部" → 自以为是
- "你看到我上一封邮件了吗？" → 施加内疚感
- "跟进我之前的消息" → 泛泛而谈，毫无增益

## 根据职级调整 CTA

**高管/创始人：** 超低努力、好奇心驱动。"好奇吗？"或"值得花 2 分钟吗？"

**中层管理者：** 更具体的价值。"想让我演示一下 [公司] 如何每周节省 15 小时吗？"

在组织架构中职位越高，你能要求的摩擦就越少。
