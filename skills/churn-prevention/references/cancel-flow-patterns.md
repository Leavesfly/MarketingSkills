# 取消流程模式

按业务类型、计费提供商和行业划分的详细取消流程模式。

---

## 按业务类型划分的取消流程

### B2C / 自助式 SaaS

高体量、低接触。流程必须在无人工干预的情况下运行。

**流程结构：**
```
Cancel button → Exit survey (1 question) → Dynamic offer → Confirm → Post-cancel
```

**特点：**
- 完全自动化，无人工介入
- 快速 — 最多 2-3 个屏幕
- 一个优惠 + 一个备选方案，而非多个选项的菜单
- 移动端优化（相当比例的取消发生在移动端）
- 每个步骤都有清晰的"继续取消"选项

**典型留存率：** 20-30%

**$29/月生产力应用的示例流程：**
1. "主要原因是什么？" → 6 个选项
2. 选择"太贵了" → "享受 3 个月 75 折优惠（节省 $21.75）"
3. 拒绝 → "或切换到我们的 Starter 计划，每月 $12"
4. 拒绝 → "很遗憾看到您离开。您的访问权限将持续到 [日期]。"

---

### B2B / 团队计划

体量较低、风险较高。个人联系是值得投入成本的。

**流程结构：**
```
Cancel button → Exit survey → Offer (or route to CS) → Confirm → Post-cancel
```

**特点：**
- 将高于 MRR 阈值的账户路由给客户成功团队
- 显示团队影响（"您的 8 名团队成员将失去访问权限"）
- 为企业账户提供管理员对管理员的电话沟通
- 更长的考虑期 — 允许"安排电话会议"作为留存选项
- 要求管理员/所有者角色才能取消（而非任何团队成员）

**典型留存率：** 30-45%（由于人工接触，留存率更高）

**基于 MRR 的路由：**

| 账户 MRR | 取消流程 |
|-------------|-------------|
| <$100/月 | 带优惠的自动化流程 |
| $100-$500/月 | 自动化 + 标记供客户成功跟进 |
| $500-$2,000/月 | 在取消完成前路由给客户成功 |
| $2,000+/月 | 阻止自助取消，需要客户成功电话沟通 |

---

### Freemium / 免费转付费

用户从付费取消返回免费层级。心理不同 — 他们不是离开，而是降级。

**流程结构：**
```
Cancel button → "Switch to Free?" prompt → Exit survey (if still cancelling) → Offer → Confirm
```

**特点：**
- 首先提供免费层级作为第一选项（而非留存优惠）
- 显示他们在免费版本中保留什么以及失去什么
- "留存"是让他们留在免费版，而不是完全失去他们
- 跟踪免费层级用户以进行未来的重新升级活动

## 按计费周期划分的取消流程

### 月度订阅用户

- 对价格更敏感，承诺期更短
- 折扣优惠效果良好（2-3 个月享 20-30% 折扣）
- 暂停功能有效（1-2 个月）
- 建议以折扣价切换到年度计划作为替代方案

**优惠优先级：**
1. 折扣（如果原因 = 价格）
2. 暂停（如果原因 = 未使用 / 临时性）
3. 年度计划切换（如果活跃但对价格敏感）

### 年度订阅用户

- 承诺度更高，通常因更强的理由而取消
- 按比例退款预期很重要
- 更长的留存窗口（他们已经付款）
- 个人联系更有理由（更高的 LTV 处于风险中）

**优惠优先级：**
1. 暂停剩余期限（如果是临时性的）
2. 计划调整 + 下次续费的积分
3. 客户成功的个人联系
4. 部分退款 + 降级（比全额退款 + 取消更好）

**退款处理：**
- 如果剩余时间较长，提供按比例退款
- 如果剩余不到 3 个月，"暂停至续费"
- 慷慨一些 — 糟糕的退款体验会制造大声抱怨的反对者

## 留存优惠模式

### 折扣阶梯

不要一开始就提供最大折扣。逐步升级：

```
Cancel click → 15% off → Still cancelling → 25% off → Still cancelling → Let them go
```

**规则：**
- 每次取消会话最多提供 2 次折扣优惠
- 绝不超过 30%（更高会训练用户为折扣而取消的行为）
- 限时折扣（2-3 个月，然后恢复全价）
- 跟踪折扣接受者 — 如果他们在全价时再次取消，不再提供优惠

### 暂停策略手册

暂停通常比折扣更好，因为它不会贬低您的产品价值。

**实施：**

| 设置 | 建议 |
|---------|---------------|
| 暂停时长选项 | 1 个月、2 个月、3 个月 |
| 默认选择 | 1 个月（最短） |
| 最长暂停 | 3 个月（更长的暂停很少会回归） |
| 暂停期间 | 保留数据，移除访问权限 |
| 重新激活 | 提前 7 天发送邮件自动重新激活 |
| 重复暂停 | 每 12 个月允许 1 次暂停 |

**暂停重新激活序列：**
- 第 -7 天："您的暂停将在 7 天后结束。我们一直在忙碌 — 以下是新功能。"
- 第 -1 天："明天欢迎回来！以下是等待您的内容。"
- 第 0 天："您回来了！以下是新功能的快速导览。"

### 降级路径

对于多计划产品，降级是最强的留存方式：

```
┌─────────────────────────────────────────┐
│  Before you go, what about right-sizing │
│  your plan?                             │
│                                         │
│  Current: Pro ($49/mo)                  │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │ Switch to Starter ($19/mo)      │    │
│  │                                 │    │
│  │ ✓ Keep: Projects, integrations  │    │
│  │ ✗ Lose: Advanced analytics,     │    │
│  │         team features           │    │
│  │                                 │    │
│  │ [Switch to Starter]             │    │
│  └─────────────────────────────────┘    │
│                                         │
│  [No thanks, continue cancelling]       │
└─────────────────────────────────────────┘
```

**降级最佳实践：**
- 准确显示他们保留什么和失去什么
- 使用对勾和叉号以提高可扫描性
- 即使在较低计划上也保留他们的数据
- 如果他们降级，至少 30 天内不显示升级提示

### 竞争对手切换处理

当取消原因是"切换到竞争对手"时：

1. **询问是哪个竞争对手**（可选，不强制）
2. **显示比较**（如果您有的话）（参见 competitor-alternatives 技能）
3. **提供迁移积分**（"我们将为您匹配他们的价格 3 个月"）
4. **请求反馈电话**（"15 分钟了解我们缺少的内容"）

这些数据对产品和营销团队来说非常宝贵。

## 取消后体验

取消后发生的事情对以下方面很重要：
- 赢回潜力
- 口碑
- 评论情感

### 确认页面

```
Your subscription has been cancelled.

What happens next:
• Your access continues until [billing period end date]
• Your data will be preserved for 90 days
• You can reactivate anytime from your account settings

[Reactivate My Account]

We'd love to have you back. We'll keep improving based on feedback
from customers like you.
```

### 取消后序列

| 时间 | 操作 |
|--------|--------|
| 立即 | 包含访问结束日期的确认邮件 |
| 第 1 天 | （无 — 不要显得急切） |
| 第 7 天 | 关于整体体验的 NPS/满意度调查 |
| 第 30 天 | 包含最近改进的"新功能"邮件 |
| 第 60 天 | 如果已解决，针对其特定取消原因进行处理 |
| 第 90 天 | 最终赢回并提供特别优惠 |

**详细的赢回邮件序列**：参见 email-sequence 技能。

## 细分规则

最有效的取消流程使用细分来向不同客户展示不同的优惠。

### 细分维度

| 维度 | 重要性 |
|-----------|---------------|
| 计划 / MRR | 高价值客户获得个人联系 |
| 任期 | 长期客户获得更慷慨的优惠 |
| 使用水平 | 高使用量客户与休眠客户获得不同的消息 |
| 计费周期 | 月度和年度需要不同的方法 |
| 之前的留存 | 不要向重复取消者重新提供相同的折扣 |
| 取消原因 | 驱动显示哪个优惠（核心映射） |

### 针对细分的流程

**新客户（< 30 天）：**
- 他们尚未激活。留存在于引导，而非折扣。
- 优惠：免费引导电话、设置帮助、延长试用期
- 询问："您希望实现什么目标？"（了解缺少的内容）

**因价格取消的活跃客户：**
- 他们喜欢产品但无法证明成本合理。
- 优惠：折扣、年度计划切换、降级
- 高留存潜力

**休眠客户（30+ 天未登录）：**
- 他们忘记了您。折扣不会让他们回来。
- 优惠：暂停订阅、"发生了什么变化？"对话
- 低留存潜力 — 专注于了解原因

**切换到竞争对手的重度用户：**
- 他们正在积极选择其他产品。
- 优惠：竞争性匹配、反馈电话、路线图预览
- 中等留存潜力 — 取决于原因

## 实施清单

### 第一阶段：基础（第 1 周）
- [ ] 添加取消流程（调查 + 1 个优惠 + 确认）
- [ ] 设置包含 5-7 个原因类别的退出调查
- [ ] 为每个原因映射一个优惠（简单的 1:1 映射）
- [ ] 在分析中跟踪取消原因和留存率
- [ ] 启用预催缴卡过期邮件

### 第二阶段：优化（第 2-4 周）
- [ ] 添加备选优惠（每个原因的主要 + 次要）
- [ ] 实施暂停订阅选项
- [ ] 设置催缴邮件序列（10 天内 4 封邮件）
- [ ] 启用智能重试（Stripe Smart Retries 或等效方案）
- [ ] 为高价值账户启用基于 MRR 的路由

### 第三阶段：高级（第 2 个月及以后）
- [ ] 从使用信号构建健康评分
- [ ] 设置主动干预触发器
- [ ] A/B 测试折扣金额和优惠类型
- [ ] 按计划、任期和使用情况细分流程
- [ ] 取消后赢回序列（与 email-sequence 技能协调）
- [ ] 队列分析：按渠道、计划、任期划分的流失

## 合规说明

### FTC Click-to-Cancel 规则（美国）
- 取消必须与注册一样简单
- 如果在线注册，不能要求通过电话取消
- 不能添加过多步骤以阻止取消
- 允许提供留存优惠，但"继续取消"必须清晰

### GDPR / 数据保留（欧盟）
- 告知用户取消后的数据保留期
- 在账户删除前提供数据导出
- 在 30 天内遵守删除请求
- 未经同意，不使用取消后的数据进行营销

### 通用最佳实践
- 始终显示清晰的完成取消路径
- 绝不隐藏取消按钮（黑暗模式）
- 即使留存流程出错也要处理取消
- 通过电子邮件收据确认取消
