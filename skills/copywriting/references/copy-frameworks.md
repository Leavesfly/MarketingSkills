**{达成理想结果}通过{产品如何实现}**

## 落地页部分类型

### 核心部分

**首屏（Above the Fold）**
- 标题 + 副标题
- 主要 CTA
- 支持性视觉元素（产品截图、主图）
- 可选：社会证明栏

**社会证明栏**
- 客户标志（可识别的 > 数量多的）
- 关键指标（"10,000+ 团队"）
- 星级评分和评论数量
- 简短推荐语片段

**问题/痛点部分**
- 比他们自己更好地阐述他们的问题
- 创造认同感（"这正是我的情况"）
- 暗示不解决它的代价

**解决方案/利益部分**
- 从问题过渡到你的解决方案
- 3-5 个关键利益（不是 10 个）
- 每个：标题 + 解释 + 如有证明则添加

**工作原理**
- 3-4 个编号步骤
- 降低感知复杂度
- 每步：行动 + 结果

**最终 CTA 部分**
- 重述价值主张
- 重复主要 CTA
- 风险逆转（保证、免费试用）

---

### 支持部分

**推荐语**
- 带有姓名、职位、公司的完整引言
- 尽可能包含照片
- 具体结果胜于模糊赞美
- 格式：引言卡片、视频、推文嵌入

**案例研究**
- 问题 → 解决方案 → 结果
- 具体指标和结果
- 客户姓名和背景
- 可以是带有"阅读更多"链接的片段

**用例**
- 产品的不同使用方式
- 帮助访客自我识别
- "面向需要 X 的营销人员"格式

**人物角色 / "专为"部分**
- 明确点名目标受众
- "完美适合[角色]"块
- 解决"这适合我吗？"的问题

**FAQ 部分**
- 解决常见异议
- 对 SEO 有益
- 减少支持负担
- 5-10 个最常见的问题

**比较部分**
- 与竞争对手比较（点名或不点名）
- 与现状比较（电子表格、手动流程）
- 表格或并排格式

**集成 / 合作伙伴**
- 你连接的工具的标志
- "与你的技术栈兼容"信息
- 建立可信度

**创始人故事 / 宣言**
- 你为何构建这个
- 你相信什么
- 情感连接
- 与无面孔的竞争对手区分开来

**演示 / 产品导览**
- 交互式演示
- 视频 walkthrough
- GIF 预览
- 展示产品实际效果

**定价预览**
- 即使在非定价页面上也提供 teaser
- 起始价格或"从 $X/月起"
- 推动决策者前进

**保证 / 风险逆转**
- 退款保证
- 免费试用条款
- "随时取消"
- 减少摩擦

**数据部分**
- 建立可信度的关键指标
- "10,000+ 客户"
- "4.9/5 评分"
- "为客户节省了 200 万美元"

## 页面结构模板

### 功能密集型页面（弱）

```
1. 首屏
2. 功能 1
3. 功能 2
4. 功能 3
5. 功能 4
6. CTA
```

这是一个列表，而不是有说服力的叙述。

---

### 多样化、引人入胜的页面（强）

```
1. 具有清晰价值主张的首屏
2. 社会证明栏（标志或数据）
3. 问题/痛点部分
4. 工作原理（3 步）
5. 关键利益（2-3 个，不是 10 个）
6. 推荐语
7. 用例或人物角色
8. 与替代方案比较
9. 案例研究片段
10. FAQ
11. 带有保证的最终 CTA
```

这讲述了一个故事并解决了异议。

---

### 紧凑型落地页

```
1. 首屏（标题、副标题、CTA、图片）
2. 社会证明栏
3. 3 个带图标的关键利益
4. 推荐语
5. 工作原理（3 步）
6. 带有保证的最终 CTA
```

适用于重视简洁性的广告落地页。

---

### 企业级/B2B 落地页

```
1. 首屏（结果导向标题）
2. 标志栏（知名公司）
3. 问题部分（业务痛点）
4. 解决方案概述
5. 按角色/部门划分的用例
6. 安全/合规部分
7. 集成标志
8. 带有指标的案例研究
9. ROI/价值部分
10. 联系/演示 CTA
```

解决企业买家关切。

---

### 产品发布页面

```
1. 带有发布公告的首屏
2. 视频演示或 walkthrough
3. 功能亮点（3-5 个）
4. 前后对比
5. 早期推荐语
6. 发布定价或早期访问优惠
7. 带有紧迫感的 CTA
```

适用于 ProductHunt、发布或公告。

---

## 部分写作技巧

### 问题部分

以以下短语开头：
- "你知道那种感觉..."
- "如果你像大多数[角色]一样..."
- "每天，[受众]都在与...斗争"
- "我们都经历过..."

然后描述：
- 具体的挫折
- 浪费的时间/金钱
- 对他们工作/生活的影响

### 利益部分

对于每个利益，包括：
- **标题**：他们获得的结果
- **正文**：如何运作（1-2 句话）
- **证明**：数字、推荐语或示例（可选）

### 工作原理部分

每步应该是：
- **编号**：创造进展感
- **简单动词**："连接"、"设置"、"获取"
- **结果导向**：他们从这一步获得什么

示例：
1. 连接你的工具（只需 2 分钟）
2. 设置你的偏好
3. 每周一获取自动报告

### 推荐语选择

最佳推荐语包括：
- 具体结果（"转化率提高了 32%"）
- 前后背景（"我们过去花费数小时..."）
- 角色 + 公司以建立可信度
- 可引用且具体的内容

避免只说以下内容的推荐语：
- "很棒的产品！"
- "喜欢它！"
- "易于使用！"
