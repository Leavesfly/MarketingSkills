# 竞争对手页面的部分模板

竞争对手比较页面每个部分的即用型模板。

## 目录
- TL;DR 摘要
- 段落比较（不仅仅是表格）
- 功能比较部分
- 定价比较部分
- 服务和支持比较
- 适合谁部分
- 迁移部分
- 社会证明部分
- 比较表最佳实践（超越复选框、按类别组织、在有用时包括评分）

## TL;DR 摘要

在每个页面开头为扫描者提供一个快速摘要：

```markdown
**TL;DR**: [Competitor] 擅长 [优势]，但在 [劣势] 方面存在困难。
[Your product] 专为 [您的重点] 而构建，提供 [关键差异化因素]。
如果 [他们的理想用例]，请选择 [Competitor]。如果 [您的理想用例]，请选择 [您]。
```

## 段落比较（不仅仅是表格）

对于每个主要维度，写一段：

```markdown
## 功能

[Competitor] 提供 [他们的方法描述]。
他们的优势是 [具体优势]，这适用于 [用例]。
然而，[局限性] 可能对 [用户类型] 具有挑战性。

[Your product] 采用不同的方法，即 [您的方法]。
这意味着 [好处]，尽管 [诚实的权衡]。
需要 [特定需求] 的团队通常发现这更有效。
```

## 功能比较部分

超越复选框：

```markdown
## 功能比较

### [功能类别]

**[Competitor]**：[2-3 句话描述他们如何处理此问题]
- 优势：[具体]
- 局限性：[具体]

**[Your product]**：[2-3 句话描述]
- 优势：[具体]
- 局限性：[具体]

**底线**：如果 [场景]，请选择 [Competitor]。如果 [场景]，请选择 [您]。
```

## 定价比较部分

```markdown
## 定价

| | [Competitor] | [Your Product] |
|---|---|---|
| 免费层 | [详情] | [详情] |
| 起始价格 | $X/user/mo | $X/user/mo |
| 业务层 | $X/user/mo | $X/user/mo |
| 企业版 | 定制 | 定制 |

**包含内容**：[Competitor] 的 $X 计划包括 [功能]，而
[Your product] 的 $X 计划包括 [功能]。

**总成本考虑**：除了每席位定价外，还要考虑 [隐藏成本、
附加组件、实施]。[Competitor] 对 [X] 额外收费，而
[Your product] 在基础定价中包含 [Y]。

**价值比较**：对于一个 10 人团队，[Competitor] 的成本约为
$X/年，而 [Your product] 的成本为 $Y/年，[您在获得的内容方面的关键差异]。
```

## 比较表最佳实践

### 超越复选框

而不是：
| Feature | You | Competitor |
|---------|-----|-----------|
| Feature A | ✓ | ✓ |
| Feature B | ✓ | ✗ |

这样做：
| Feature | You | Competitor |
|---------|-----|-----------|
| Feature A | Full support with [detail] | Basic support, [limitation] |
| Feature B | [Specific capability] | Not available |

### 按类别组织

将功能分组为有意义的类别：
- 核心功能
- 协作
- 集成
- 安全和合规
- 支持和服务

### 在有用时包括评分

| Category | You | Competitor | Notes |
|----------|-----|-----------|-------|
| Ease of use | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | [Brief note] |
| Feature depth | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | [Brief note] |
