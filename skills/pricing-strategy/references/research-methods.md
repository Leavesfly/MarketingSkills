## Van Westendorp 价格敏感度测试

Van Westendorp 调查确定你产品的可接受价格范围。

### 四个问题

询问每位受访者：
1. "你认为 [产品] 的价格高到什么程度，以至于你不会考虑购买？"（太贵）
2. "你认为 [产品] 的价格低到什么程度，以至于你会质疑其质量？"（太便宜）
3. "你认为 [产品] 的价格开始变得昂贵，但你仍可能会考虑？"（昂贵/高价侧）
4. "你认为 [产品] 的价格划算——物超所值？"（便宜/高性价比）

### 如何分析

1. 绘制每个问题的累积分布
2. 找到交叉点：
   - **边际便宜点 (PMC)：** "太便宜"与"昂贵"交叉
   - **边际昂贵点 (PME)：** "太贵"与"便宜"交叉
   - **最优价格点 (OPP)：** "太便宜"与"太贵"交叉
   - **无差异价格点 (IDP)：** "昂贵"与"便宜"交叉

**可接受的价格范围：** PMC 到 PME
**最佳定价区间：** OPP 和 IDP 之间

### 调查提示
- 需要 100-300 名受访者才能获得可靠数据
- 按角色细分（不同的支付意愿）
- 使用现实的产品描述
- 考虑添加购买意向问题

### 示例输出

```
价格敏感度分析结果：
─────────────────────────────────
边际便宜点：  $29/月
最优价格点：          $49/月
无差异价格点：     $59/月
边际昂贵点： $79/月

推荐范围：$49-59/月
当前价格：$39/月（低于最优）
机会：价格可提高 25-50% 而不会对需求产生显著影响
```

## MaxDiff 分析（最佳-最差缩放）

MaxDiff 识别客户最重视的功能，为包装决策提供信息。

### 工作原理

1. 列出 8-15 个你可以包含的功能
2. 向受访者展示每组 4-5 个功能
3. 问："哪个最重要？哪个最不重要？"
4. 在多个组中重复，直到所有功能都被比较
5. 统计分析产生重要性评分

### 示例调查问题

```
哪个功能对你来说最重要？
哪个功能对你来说最不重要？

□ 无限项目
□ 自定义品牌
□ 优先支持
□ API 访问
□ 高级分析
```

### 分析结果

功能按效用评分排名：
- 高效用 = 必备（包含在基础层级中）
- 中等效用 = 差异化因素（用于层级分离）
- 低效用 = 锦上添花（高级层级或削减）

### 将 MaxDiff 用于包装

| 效用评分 | 包装决策 |
|---------------|-------------------|
| 前 20% | 包含在所有层级中（基本要求） |
| 20-50% | 用于区分层级 |
| 50-80% | 仅更高层级 |
| 后 20% | 考虑削减或作为高级附加组件 |

## 支付意愿调查

**直接方法（简单但有偏差）：**
"你愿意为 [产品] 支付多少钱？"

**更好：Gabor-Granger 方法：**
"你会以 [$X] 的价格购买 [产品] 吗？"（是/否）
在不同受访者之间变化价格以构建需求曲线。

**甚至更好：联合分析：**
展示不同价格的产品捆绑包
受访者选择首选选项
统计分析揭示每个功能的价格敏感度

## 用量-价值相关性分析

### 1. 检测用量数据
跟踪客户如何使用你的产品：
- 功能使用频率
- 用量指标（用户、记录、API 调用）
- 结果指标（产生的收入、节省的时间）

### 2. 与客户成功相关联
- 哪些使用模式预测留存？
- 哪些使用模式预测扩展？
- 哪些客户支付最多，为什么？

### 3. 识别价值阈值
- 在什么用量水平下客户"理解了"？
- 在什么用量水平下他们会扩展？
- 在什么用量水平下应该提高价格？

### 示例分析

```
用量-价值相关性分析：
─────────────────────────────────
细分：高 LTV 客户（>$10k ARR）
平均每月活跃用户：15
平均项目数：8
平均集成数：4

细分：流失客户
平均每月活跃用户：3
平均项目数：2
平均集成数：0

洞察：价值与团队采用率（用户）
        和使用深度（集成）相关

建议：按用户定价，将集成限制在更高层级
```
