## 多少个层级？

**2 个层级：** 简单、清晰的选择
- 适用于：清晰的中小企业与企业级区分
- 风险：可能会损失收入

**3 个层级：** 行业标准
- 好层级 = 入口点
- 更好层级 = 推荐（锚定到最佳）
- 最佳层级 = 高价值客户

**4+ 个层级：** 更精细
- 适用于：广泛的客户规模
- 风险：决策瘫痪、复杂性

## 好-更好-最佳框架

**好层级（入门）：**
- 目的：消除进入障碍
- 包含：核心功能、有限用量
- 价格：低、易于访问
- 目标：小团队、先试后买

**更好层级（推荐）：**
- 目的：大多数客户落脚点
- 包含：完整功能、合理限制
- 价格：你的"锚定"价格
- 目标：成长中的团队、认真用户

**最佳层级（高级）：**
- 目的：捕获高价值客户
- 包含：所有内容、高级功能、更高限制
- 价格：溢价（通常是"更好"的 2-3 倍）
- 目标：更大团队、高级用户、企业级

## 层级差异化策略

**功能门控：**
- 所有层级中的基础功能
- 更高层级中的高级功能
- 适用于功能具有明确价值差异的情况

**用量限制：**
- 相同功能、不同限制
- 更高层级中更多用户、存储、API 调用
- 适用于价值随用量扩展的情况

**支持级别：**
- 邮件支持 → 优先支持 → 专属成功经理
- 适用于实施复杂的产品

**访问和定制：**
- API 访问、SSO、自定义品牌
- 适用于企业级差异化

## 基于角色的包装

### 识别定价角色

不同的客户有不同的：
- 支付意愿
- 功能需求
- 购买流程
- 价值感知

**细分依据：**
- 公司规模（自由职业者 → 中小企业 → 企业级）
- 用例（营销 vs. 销售 vs. 支持）
- 熟练程度（初学者 → 高级用户）
- 行业（不同的预算规范）

### 基于角色的包装

**步骤 1：定义角色**

| 角色 | 规模 | 需求 | WTP | 示例 |
|---------|------|-------|-----|---------|
| 自由职业者 | 1 人 | 基础功能 | 低 | $19/月 |
| 小团队 | 2-10 | 协作 | 中等 | $49/月 |
| 成长公司 | 10-50 | 扩展、集成 | 较高 | $149/月 |
| 企业级 | 50+ | 安全、支持 | 高 | 自定义 |

**步骤 2：将功能映射到角色**

| 功能 | 自由职业者 | 小团队 | 成长型 | 企业级 |
|---------|------------|------------|---------|------------|
| 核心功能 | ✓ | ✓ | ✓ | ✓ |
| 协作 | — | ✓ | ✓ | ✓ |
| 集成 | — | 有限 | 完整 | 完整 |
| API 访问 | — | — | ✓ | ✓ |
| SSO/SAML | — | — | — | ✓ |
| 审计日志 | — | — | — | ✓ |
| 自定义合同 | — | — | — | ✓ |

**步骤 3：为每个角色按价值定价**
- 研究每个细分市场的支付意愿
- 设定能够捕获价值而不阻碍采用的价格
- 考虑针对特定细分市场的落地页

## 免费增值与免费试用

### 何时使用免费增值

**免费增值适用于：**
- 产品具有病毒式/网络效应
- 免费用户提供价值（内容、数据、推荐）
- 大市场，其中 % 转化驱动体量
- 服务免费用户的边际成本低
- 明确的升级触发功能/用量限制

**免费增值风险：**
- 免费用户可能永远不会转化
- 贬低产品感知
- 非付费用户的支持成本
- 以后更难提高价格

### 何时使用免费试用

**免费试用适用于：**
- 产品需要时间来展示价值
- 需要入职/设置投资
- B2B 且有购买委员会
- 较高的价格点
- 产品一旦配置就具有"粘性"

**试用最佳实践：**
- 简单产品 7-14 天
- 复杂产品 14-30 天
- 完全访问（非功能限制）
- 清晰的倒计时和提醒
- 是否需要信用卡的权衡

** upfront 信用卡：**
- 更高的试用转付费转化率（40-50% 对比 15-25%）
- 更低的试用量
- 更好的合格线索

### 混合方法

**免费增值 + 试用：**
- 功能有限的免费层级
- 高级功能的试用
- 示例：Zoom（免费 40 分钟，Pro 试用）

**反向试用：**
- 从完全访问开始
- 试用后降级到免费层级
- 示例：看到高级价值，在准备好之前忍受限制

## 企业级定价

### 何时添加自定义定价

当以下情况时添加"联系销售"：
- 交易规模超过 $10k+ ARR
- 客户需要自定义合同
- 需要实施/入职
- 安全/合规要求
- 涉及采购流程

### 企业级层级元素

**基本要求：**
- SSO/SAML
- 审计日志
- 管理员控制
-  uptime SLA
- 安全认证

**增值项：**
- 专属支持/成功经理
- 自定义入职
- 培训课程
- 自定义集成
- 优先路线图输入

### 企业级定价策略

**大规模按席位：**
- 大团队的批量折扣
- 示例：$15/用户（标准）→ $10/用户（100+）

**平台费 + 用量：**
- 访问的基础费用
- 超过阈值的基于用量的费用
- 示例：$500/月基础 + 每次 API 调用 $0.01

**基于价值的合同：**
- 价格与客户收入/结果挂钩
- 示例：交易百分比、收入分成
