# ASO 评分标准

使用以下评分细则对每个维度进行 0-10 分评分。
**应用主技能第 1.5 阶段中的品牌成熟度层级调整**。

---

## 品牌成熟度调整（应用于所有维度）

在评分前，确定应用的层级：**主导型**、**成熟型**或**挑战者型**。

**主导型应用（Instagram、Uber、Spotify、WhatsApp、Netflix）：**

- 纯品牌标题得分 8+（品牌本身就是关键词）
- 生活方式/品牌截图与带说明的 UI 截图得分相同
- 每周或更频繁更新的通用"What's New"得分 8+
- 实用类应用缺少应用内活动不扣分
- 描述仅按转化质量评分，不按关键词存在与否评分
- 本地化根据实际市场覆盖范围评分
- 如果品牌知名度接近普及，缺少预览视频可以接受

**成熟型应用（Duolingo、Strava、Notion、Calm、Cash App）：**

- 品牌优先标题加 1-2 个关键词正常评分
- 战略性描述/视觉选择给予合理怀疑
- 所有其他维度正常评分

**挑战者型应用（大多数应用）：**

- 严格按照教科书式 ASO 评分 — 每个字符和功能都很重要

**关键原则：** 在扣分之前，问自己："这是一个错误，还是拥有比我更多信息的团队做出的数据驱动决策？"

---

## 1. 标题与副标题（权重：20%）

**挑战者型评分细则：**

| 分数 | 标准                                                                                                                                                                |
| ----- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 9-10  | 标题包含品牌 + 高价值关键词，副标题包含互补关键词，字段间无词语重复，接近最大字符数限制，立即传达应用目的 |
| 7-8   | 良好的关键词存在，轻微字符浪费（5+ 未用字符），目的清晰                                                                                           |
| 5-6   | 有关键词但位置不佳，字段间有重复，目的大致清晰                                                                                 |
| 3-4   | 标题为纯品牌或通用，副标题缺失或薄弱，字符使用差                                                                                          |
| 1-2   | 无关键词策略，标题不传达目的，严重字符浪费                                                                                           |
| 0     | 无法评估（数据不可用）                                                                                                                                        |

**主导型/成熟型调整：** 如果品牌具有高搜索量，纯品牌标题（例如"Instagram"）是有效的。对于品牌认知度消除通用关键词需求的主导型应用，得分 8+。评估未用字符是代表浪费还是有意简化。

**检查项：**

- 已用字符 vs 限制（标题：30，副标题/简短描述：30/80）。"接近最大" = 距限制 3 个字符以内（27+/30，77+/80）
- 标题中包含主要关键词
- 标题和副标题之间的关键词重复
- 应用目的是否立即清晰
- 不必要的词语（冠词、介词）占用空间
- 可能导致拒绝的特殊字符或声明（"#1"、"最佳"）（Apple）

---

## 2. 描述（权重：15%）

### Apple App Store

| 分数 | 标准                                                                                                                                               |
| ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 9-10  | 前 3 行以清晰的价值主张吸引用户，结构包含功能/好处/社会证明/行动号召，积极使用推广文本，引人注目且易于扫描 |
| 7-8   | 良好的开头，合理的结构，可改进易读性或行动号召                                                                                      |
| 5-6   | 通用开头（"欢迎来到..."），有一些结构，缺少行动号召或社会证明                                                                         |
| 3-4   | 大段文字，首屏无清晰价值主张，无推广文本                                                                                      |
| 1-2   | 最小化或模板化描述，无需努力                                                                                                          |
| 0     | 无法评估                                                                                                                                          |

### Google Play

| 分数 | 标准                                                                                                                                     |
| ----- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| 9-10  | 前 3 句包含关键词，全文自然密度 2-3%，使用 HTML 格式，结构化章节，强行动号召，关键词感觉自然 |
| 7-8   | 良好的关键词存在，有一些结构，密度略有偏差（1-2% 或 3-4%）                                                                   |
| 5-6   | 关键词存在但稀疏（<1%）或堆砌（>5%），结构弱                                                                           |
| 3-4   | 无可见关键词策略，格式差，大段文字                                                                                   |
| 1-2   | 最小化描述，无关键词，无结构                                                                                               |
| 0     | 无法评估                                                                                                                                |

**检查项：**

- 前 3 行质量（"阅读更多"之前可见）
- 功能-好处框架（不仅仅是功能列表）
- 社会证明（下载量、奖项、媒体报道）
- 行动号召
- 关键词密度（仅 Google Play - 计算目标关键词数 / 总词数）
- HTML 格式使用（Google Play）
- 推广文本存在性和质量（Apple）

---

## 3. 视觉素材（权重：25%）

| 分数 | 标准                                                                                                                                                                      |
| ----- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 9-10  | 8-10 张截图带有清晰消息/说明，有预览视频，截图按顺序讲述故事，每张传达一个好处，图标独特且令人难忘 |
| 7-8   | 6-7 张带说明的截图，良好图标，无视频或有良好视频但某些截图消息不清晰                                                                        |
| 5-6   | 5+ 张截图但说明弱/无，基础图标，无视频，截图是 UI 转储                                                                                           |
| 3-4   | 3-4 张截图，无说明，通用图标，无故事性                                                                                                                   |
| 1-2   | 少于 3 张截图，或截图是原始未编辑的 UI，图标差                                                                                                       |
| 0     | 无法评估                                                                                                                                                                 |

**检查项：**

- 截图数量（最少 5 张，理想 8-10 张）
- 截图上的说明/叠加文本（每屏一条消息，最多 5-7 个词）
- 前 3 张截图（Apple 上转化影响最高）
- 预览视频存在性和质量
- 图标独特性（图标中无文本，大胆形状，脱颖而出）
- 功能图形存在（Google Play - 推荐位必需）
- 截图故事流（它们是否讲述连贯的故事？）
- 本地化视觉素材（针对非英语市场）
- 说明关键词（Apple - 自 2025 年 6 月起索引）

---

## 4. 评分与评论（权重：20%）

| 分数 | 标准                                                                                               |
| ----- | ------------------------------------------------------------------------------------------------------ |
| 9-10  | 4.5+ 星，10K+ 评分，近期评论正面，开发者回复负面评论，稳定的评论流 |
| 7-8   | 4.0-4.4 星，1K+ 评分， mostly 正面近期评论，一些开发者回复                   |
| 5-6   | 3.5-3.9 星，500+ 评分，混合近期评论，无开发者回复                              |
| 3-4   | 3.0-3.4 星，<500 评分，近期评论中有负面主题                                         |
| 1-2   | 低于 3.0 星，少量评分，无开发者参与，可见投诉                              |
| 0     | 尚无评分或无法评估                                                                        |

**检查项：**

- 平均评分（目标：最低 4.0+，理想 4.5+）
- 总评分数
- 近期评论情感（最后 5-10 条可见评论）
- 常见投诉主题（bug、崩溃、定价、UX）
- 开发者回复存在性和质量
- 评分趋势（改善或下降，如果可见）
- 评论新鲜度（新鲜评论表明活跃用户群）

---

## 5. 元数据与新鲜度（权重：10%）

| 分数 | 标准                                                                                                                  |
| ----- | ------------------------------------------------------------------------------------------------------------------------- |
| 9-10  | 上月内更新，10+ 本地化，最佳类别选择，应用内活动/LiveOps 活跃，数据安全完整 |
| 7-8   | 2 个月内更新，5+ 本地化，良好类别，存在数据安全                                             |
| 5-6   | 3 个月内更新，2-4 本地化，可接受类别                                                           |
| 3-4   | 3-6 个月前更新，1-2 本地化，可能类别错误                                                        |
| 1-2   | 6+ 个月未更新，单语言，类别选择差                                                           |
| 0     | 无法评估                                                                                                             |

**检查项：**

- 最后更新日期和新鲜度
- 支持的语言/本地化数量
- 类别选择（是否是最佳匹配？竞争较少的替代方案？）
- 应用内活动（Apple）或推广内容（Google）存在
- 数据安全/隐私营养标签完整性
- 年龄评级适当性
- 版本历史质量（发布说明是否传达价值？）
- What's New 文本质量

---

## 6. 转化信号（权重：10%）

| 分数 | 标准                                                                                                                                                          |
| ----- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 9-10  | 下载前价值清晰，定价/IAP 透明，可见社会证明（媒体报道、奖项），下载范围表明强劲 traction，开发者可信度强 |
| 7-8   | 良好的价值传达，定价清晰，一些社会证明                                                                                                        |
| 5-6   | 价值主张存在但弱，定价不清晰或 IAP 重，有限社会证明                                                                                    |
| 3-4   | 不清楚用户得到什么，定价混乱，无社会证明，可见下载量低                                                                                 |
| 1-2   | 无价值传达，可疑定价，应用看起来被 abandoned                                                                                                   |
| 0     | 无法评估                                                                                                                                                     |

**检查项：**

- 价格透明度（免费、增值、付费 - 是否清晰？）
- 应用内购买列表质量（IAP 名称是否传达价值？）
- 下载范围（Google Play - 10K+、100K+、1M+ 表示信任）
- 开发者名称/品牌认知度
- "编辑推荐"或推荐徽章
- 描述中的媒体报道或奖项
- 同一开发者的相关应用（组合信任信号）
- 隐私实践透明度

---

## 计算最终得分

```
最终得分 = (标题 * 0.20) + (描述 * 0.15) + (视觉素材 * 0.25)
            + (评分 * 0.20) + (元数据 * 0.10) + (转化 * 0.10)

缩放到 100：最终得分 * 10
```

**示例：** 标题：7，描述：6，视觉素材：8，评分：9，元数据：5，转化：7

```
(7 * 0.20) + (6 * 0.15) + (8 * 0.25) + (9 * 0.20) + (5 * 0.10) + (7 * 0.10)
= 1.4 + 0.9 + 2.0 + 1.8 + 0.5 + 0.7
= 7.3 → 73/100 → 等级：B
```
