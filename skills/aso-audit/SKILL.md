# ASO 审计

根据 ASO 最佳实践分析 App Store 和 Google Play 应用商店列表。获取实时列表数据，对元数据、视觉效果和评分进行评分，然后生成优先行动计划。

## 何时使用

- 用户分享 App Store 或 Google Play URL
- 用户要求审计或优化应用列表
- 用户想要将其应用与竞争对手进行比较
- 用户询问应用商店排名、可见性或下载转化率

## 审计前

**首先检查产品营销上下文：**
如果存在 `.agents/product-marketing-context.md`（或在旧版设置中为 `.claude/product-marketing-context.md`），请在提问之前阅读它。使用该上下文，仅询问尚未涵盖或与此任务相关的信息。

## 第一阶段 — 识别商店并获取

### 从 URL 检测商店类型

```
Apple:  apps.apple.com/{country}/app/{name}/id{digits}
Google: play.google.com/store/apps/details?id={package}
```

如果用户提供的是应用名称而非 URL，请在网络上搜索：
`site:apps.apple.com "{app name}"` 或 `site:play.google.com "{app name}"`

### 获取列表

使用 WebFetch 检索列表页面。提取每个可用字段：

**Apple App Store 字段：**

- 应用名称（标题）— 30 字符限制
- 副标题 — 30 字符限制
- 描述（长）— 不用于搜索索引，但对转化很重要
- 促销文本 — 170 字符，无需新版本即可更新
- 类别（主要 + 次要）
- 截图（数量、顺序、说明文字）
- 预览视频（是否存在、时长）
- 评分（平均值 + 数量）
- 近期评论（可见的）
- 价格 / 应用内购买
- 开发者名称
- 最后更新日期
- 版本历史记录注释
- 年龄分级
- 大小
- 列出的语言 / 本地化
- 应用内活动（如果有可见的）

**Google Play 字段：**

- 应用名称（标题）— 30 字符限制
- 简短描述 — 80 字符限制
- 完整描述 — 4,000 字符限制，会被搜索索引
- 类别 + 标签
- 特色图形（是否存在）
- 截图（数量、顺序）
- 预览视频（是否存在）
- 评分（平均值 + 数量）
- 近期评论（可见的）
- 价格 / 应用内购买
- 开发者名称
- 最后更新日期
- "最新动态"文本
- 下载量范围
- 内容分级
- 数据安全部分
- 列出的语言

如果 WebFetch 返回的数据不完整（商店采用客户端渲染），请注明差距并利用现有数据。如果缺少关键信息，请用户粘贴缺失字段。

### 视觉素材评估

WebFetch 无法提取截图图像或说明文字。**截取列表页面的屏幕截图**以获取视觉数据：

1. 导航到列表 URL 并捕获整页截图
2. 评估截图：图标质量、截图数量、说明文字、消息质量、预览视频是否存在、特色图形（Google Play）
3. 如果浏览器工具不可用，请用户分享列表页面的截图

**促销文本（Apple）：** 这个 170 字符字段出现在描述上方，但在抓取的 HTML 中通常难以与描述区分。如果无法确认其存在，请注明并建议用户检查 App Store Connect。

## 第一阶段.5 — 评估品牌成熟度

在评分之前，将应用分类为三个层级之一。这将决定你如何解读"教科书式 ASO"的偏差——知名品牌的有意选择与不知名应用的错失机会是不同的。

### 层级定义

| 层级 | 信号 | 示例 |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------- |
| **主导型** | 家喻户晓的名字，100 万+ 评分，类别排名前 10，几乎普遍的品牌认知度。用户按品牌名称搜索，而非通用关键词。 | Instagram、Uber、Spotify、WhatsApp、Netflix |
| **成熟型** | 在其类别中知名，10 万+ 评分，强大的自然安装量，品牌被认可但并非普遍知名。 | Strava、Notion、Duolingo、Cash App、Calm |
| **挑战者型** | 正在建立知名度，<10 万评分，需要通过关键词和 ASO 策略进行发现。大多数应用属于此类。 | 你的应用、大多数独立/初创应用 |

### 层级如何影响评分

**主导型应用**在以下领域获得调整后的评分：

- **标题：** 纯品牌或品牌优先的标题是有效的（如果品牌是关键词，则评分 8+）。这些应用不需要通用关键词发现。
- **描述：** 纯粹根据转化质量评分，而非关键词存在。如果应用是家喻户晓的名字，精心打造的品牌描述胜过堆砌关键词的描述。
- **视觉素材：** 生活方式/品牌摄影而非 UI 演示是合法的转化策略。如果产品难以在 30 秒内演示或品牌知名度接近普遍，没有视频是可以接受的。
- **最新动态：** 每周或更频繁发布的通用发布说明是可以接受的（评分 8+）。在规模上，详细的变更日志 ROI 最小且可能引起反弹。
- **应用内活动：** 对于拥有庞大安装基数的实用应用（Uber、WhatsApp），缺少活动不是惩罚。这些应用不需要发现帮助。
- **本地化：** 相对于实际市场评分，而非绝对数量。仅限美国的金融科技应用有 2 种语言（英语 + 西班牙语）是适当的本地化。

**成熟型应用**获得部分调整：

- 品牌优先的标题可以接受，但仍应包含 1-2 个关键词
- 战略性描述选择获得合理怀疑的好处
- 其他维度正常评分

**挑战者型应用**严格按照教科书 ASO 最佳实践评分——每个字符、截图和关键词都很重要。

**关键原则：** 在扣分之前，请问："这是一个错误还是拥有我所没有的数据的团队做出的有意选择？"如果应用有 100 万+ 评分和专门的 ASO 团队，除非明显错误，否则假设他们的选择是基于数据的。

## 第二阶段 — 对每个维度评分

使用 `references/scoring-criteria.md` 中的标准对每个维度进行 0-10 分评分。应用第一阶段.5 中的品牌成熟度层级调整。

平台规范和基准的参考文件：

- `references/apple-specs.md` — Apple 官方字符限制、截图/视频规范、CPP/PPO 规则、拒绝触发器
- `references/google-play-specs.md` — Google Play 官方限制、截图规范、Android Vitals 阈值、政策
- `references/benchmarks.md` — 转化数据、评分影响、视频提升、截图行为、CPP/活动基准

### 维度和权重

| # | 维度 | 权重 | 涵盖内容 |
| --- | -------------------- | ------ | ------------------------------------------------------------------------- |
| 1 | 标题和副标题 | 20% | 字符使用、关键词存在、清晰度、品牌 + 关键词平衡 |
| 2 | 描述 | 15% | 前 3 行、关键词密度（Google）、CTA、结构、促销文本 |
| 3 | 视觉素材 | 25% | 截图数量/质量/消息、视频、图标、特色图形 |
| 4 | 评分和评论 | 20% | 平均评分、数量、近期性、开发者回复 |
| 5 | 元数据和新鲜度 | 10% | 类别选择、更新近期性、本地化数量、数据安全 |
| 6 | 转化信号 | 10% | 价格定位、IAP 透明度、社会证明、下载量范围 |

**最终得分** = 加权总和，满分 100。

### 评分解释

| 得分 | 等级 | 含义 |
| ------ | ----- | --------------------------------------------------------- |
| 85-100 | A | 优化良好；专注于 A/B 测试和迭代 |
| 70-84 | B | 基础良好；有明显的改进机会 |
| 50-69 | C | 存在显著差距；优先修复将产生高影响 |
| 30-49 | D | 多个维度需要重大优化 |
| 0-29 | F | 列表需要彻底改造 |

## 第三阶段 — 竞争对手比较（可选）

如果用户提供竞争对手 URL 或要求比较：

1. 获取同一类别中的 2-3 个顶级竞争对手
2. 对每个竞争对手运行相同的评分
3. 构建比较表，突出显示用户应用的弱点/优势
4. 识别关键词差距——竞争对手排名靠前但用户应用未定位的术语

如果未指定竞争对手，建议用户提供 2-3 个或提议搜索其类别中的顶级应用。

## 第四阶段 — 生成报告

使用 `references/report-template.md` 中的模板构建输出。

报告必须包括：

1. **评分卡** — 包含所有 6 个维度、得分和等级的表格
2. **前 3 个快速胜利** — 耗时 <1 小时且影响最大的更改
3. **详细调查结果** — 每个维度的细分，包含具体问题和修复方法
4. **关键词建议** — 基于标题/描述分析和竞争对手差距
5. **视觉素材建议** — 具体的截图/视频改进
6. **优先行动计划** — 按影响与努力程度排序的更改列表

### 报告规则

- 每个建议必须**具体且可操作**（"将副标题从 X 改为 Y"而非"改进副标题"）
- 包含所有文本建议的字符数
- 在相关时标记平台特定差异（Apple vs Google）
- 注明没有付费工具无法评估的内容（搜索量、确切排名）
- 建议关键词更改时，解释每个关键词为何重要

## 平台特定规则

### Apple App Store — 关键事实

- 标题（30 字符）+ 副标题（30 字符）+ 关键词字段（100 **字节**，隐藏）= 索引文本
- 关键词字段是字节而非字符——阿拉伯语/CJK 每字符使用 2-3 字节
- 长描述不用于搜索索引——仅针对转化优化
- 促销文本（170 字符）不影响搜索（Apple 已确认）
- 切勿在标题/副标题/关键词字段中重复单词（Apple 每个单词只索引一次）
- 关键词字段：逗号，无空格（"photo,editor,filter"而非"photo, editor, filter"）
- 截图：每种设备最多 10 张。前 3 张在搜索中可见——90% 的用户不会滚动到第 3 张之后
- 截图说明自 2025 年 6 月起被索引（AI 提取）
- 应用内活动：最多同时发布 10 个，每个最多 31 天。被索引并出现在搜索中
- 自定义产品页面（最多 70 个）自 2025 年 7 月起出现在有机搜索中。平均转化提升 +5.9%
- 应用预览视频：最多 3 个，每个 15-30 秒。静音自动播放——转化提升 +20-40%
- SKStoreReviewController：每 365 天最多提示 3 次
- Apple 有人工编辑策划——质量和设计更重要
- 参见 `references/apple-specs.md` 获取完整规范、尺寸和拒绝触发器

### Google Play — 关键事实

- 标题（30 字符）+ 简短描述（80 字符）+ 完整描述（4,000 字符）= 索引文本
- 完整描述会被索引——自然地目标 2-3% 关键词密度
- 没有隐藏关键词字段——所有关键词必须在可见文本中
- Google NLP/语义理解——检测到关键词堆砌并惩罚
- 标题中禁止：表情符号、全大写、"best"/"#1"/"free"、CTA（自 2021 年起执行）
- 截图：最少 2 张，**最多 8 张**每种设备（不像 Apple 的 10 张）
- 特色图形（1024x500，精确）是特色展示位置所必需的
- 视频不会自动播放——只有约 6% 的用户点击播放（与 iOS 相比 ROI 低）
- Android Vitals 直接影响排名：崩溃率 >1.09% 或 ANR >0.47% = 可见性降低
- 促销内容：提前 14 天提交以获得展示。应用看到 2 倍的探索获取量
- 自定义商店列表：最多 50 个（可以针对流失用户、特定国家、广告活动）
- 商店列表实验：测试最多 3 个变体，运行 7+ 天，一次运行 1 个实验
- 参见 `references/google-play-specs.md` 获取完整规范和政策详情

### Apple 索引 vs Google 索引

| 字段 | Apple 索引？ | Google 索引？ |
| --------------------- | ---------------- | ---------------------- |
| 标题 | 是 | 是（最强信号） |
| 副标题 / 简短描述 | 是 | 是 |
| 关键词字段 | 是（隐藏） | 不存在 |
| 长描述 | 否 | 是（大量） |
| 截图说明 | 是（自 2025 年起） | 否 |
| 应用内活动 | 是 | N/A（改用 LiveOps） |
| 开发者名称 | 否 | 部分 |
| IAP 名称 | 是 | 是 |

## 常见问题清单

如果发现这些问题，请标记。标记为 _(层级依赖)_ 的项目应根据应用的品牌成熟度层级进行评估——它们可能是主导型应用的有意选择。

**始终标记（所有层级）：**

- [ ] 评分低于 4.0
- [ ] 最后更新 > 3 个月前
- [ ] Google Play 描述没有关键词策略（密度低于 1%）
- [ ] Google Play 缺少特色图形
- [ ] Apple 关键词字段可能有重复单词（从标题+副标题推断）
- [ ] 类别不匹配——应用在另一个类别中面临的竞争较少
- [ ] 少于 5 张截图

**仅标记挑战者型/成熟型** _（对主导型应用而言不是错误）：_

- [ ] 标题仅在品牌名称上浪费字符（无关键词）_（主导型：品牌就是关键词）_
- [ ] 副标题/简短描述复制标题关键词
- [ ] 描述前 3 行是通用的 _（主导型：可能是品牌声音选择）_
- [ ] 没有预览视频 _（主导型：如果产品难以演示，可能是合理的）_
- [ ] 截图只是 UI 转储，没有消息/说明 _（主导型：生活方式/品牌照片可能转化更好）_
- [ ] 只有 1-2 种本地化 _（相对于实际市场评分，而非绝对数量）_
- [ ] 没有应用内活动或促销内容 _（主导型实用应用可能不需要发现帮助）_

**标记所有层级但注明上下文：**

- [ ] 开发者未回复负面评论 _（注意数量——在 1000 万+ 评论时回复与在 1000 条评论时回复是不同的挑战）_
- [ ] 通用的"最新动态"文本 _（对于成熟型/主导型应用的每周或更频繁发布节奏是可以接受的）_
